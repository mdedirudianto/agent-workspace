## Privacy Constraint (Design Driver)

`numa` runs `numa-home` — a bun-based Telegram AI companion managed by PM2. Its logs contain real private conversation content. **These must never be shipped to OpenObserve.**

Audit confirmed before configuring:
- `numa-home` runs under PM2, logs exclusively to `~/.pm2/logs/numa-home-{out,error}.log`
- Bun is at `/root/.bun/bin/bun` — invoked by PM2, no systemd unit
- No `openclaw.service`, `numa-home.service`, or any app-level systemd unit exists
- PM2 app stdout/stderr never enters journald — it goes to PM2 log files only

**Approach: `systemd` input with explicit whitelist.**
Fluent Bit reads journald directly and includes only the three named sources below. Everything else — including any future bun apps, OpenClaw daemon, or PM2 processes — is excluded by default. No additional config change needed to stay private as new apps are added.

## What Gets Shipped (Whitelist)

| Source | Systemd_Filter | Records seen |
|--------|---------------|-------------|
| `ssh.service` | `_SYSTEMD_UNIT=ssh.service` | Login/auth events, failed attempts, key fingerprints |
| `fail2ban.service` | `_SYSTEMD_UNIT=fail2ban.service` | IP bans and unbans |
| Kernel | `_TRANSPORT=kernel` | UFW BLOCK events, hardware events |

Verified sample from OO immediately after start:
- `ssh.service` — `Disconnected from authenticating user root 191.97.12.90 port 49490 [preauth]`
- `kernel` — `[UFW BLOCK] IN=eth0 SRC=170.64.194.156 DST=46.250.239.44 PROTO=TCP DPT=46941`

## Fluent Bit Config

**`/etc/fluent-bit/fluent-bit.conf`:**

```ini
[SERVICE]
    flush        5
    daemon       Off
    log_level    info
    parsers_file parsers.conf
    plugins_file plugins.conf

# PRIVACY NOTE: numa-home (bun/Telegram AI companion) runs via PM2.
# PM2 app logs go to ~/.pm2/logs/ — never read by Fluent Bit.
# This whitelist captures ONLY ssh, fail2ban, and kernel — nothing else.

[INPUT]
    name              systemd
    tag               numa.system
    Systemd_Filter    _SYSTEMD_UNIT=ssh.service
    Systemd_Filter    _SYSTEMD_UNIT=fail2ban.service
    Systemd_Filter    _TRANSPORT=kernel
    DB                /var/lib/fluent-bit/numa-systemd.db
    Strip_Underscores On
    Read_From_Tail    On

[FILTER]
    name   record_modifier
    match  numa.system
    record server numa
    record host   numa

[OUTPUT]
    name        http
    match       numa.system
    host        10.0.0.3
    port        5080
    uri         /api/default/numa/_json
    format      json
    http_user   admin@biji.uk
    http_passwd <redacted>
    tls         off
    compress    gzip
```

## Record Shape in OpenObserve

```json
{
  "_timestamp": 1779712606189362,
  "server": "numa",
  "host": "numa",
  "systemd_unit": "ssh.service",
  "transport": "syslog",
  "message": "Disconnected from authenticating user root 191.97.12.90 port 49490 [preauth]"
}
```

Key fields: `systemd_unit` (source unit), `transport` (`syslog` or `kernel`), `message` (log line), `server`, `host`.
