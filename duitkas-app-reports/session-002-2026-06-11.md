## Notes / conventions established

- **Per-client isolation pattern** (use for the next client too): own folder `~/miaw-flow-<client>`, own PM2 `miaw-{api,web}-<client>`, own ports (next free pair after 7019), own DB `miawflow_<client>`, own Redis db index, own `MIAW_INSTANCE_ID=<client>-bot`, own `DUITKAS_API_KEY`, private bind `10.0.0.5`, proxied at `<client>.duitkas.biji.uk` (grey-cloud DNS + LE cert, path-routed `/socket.io`+`/health`→api, `/`→web).
- `MIAW_SESSION_PATH=./data/sessions` (relative, resolved under `apps/api`) is the clean form — avoids baznas's literal-`~` directory quirk.
- New-client DB: `db:migrate` (committed migrations), **never `db:seed`** (it clears + inserts dev sample data).
- DNS `*.duitkas.biji.uk` must stay **grey-cloud (DNS-only)** → `46.250.234.153`; CF free Universal SSL doesn't cover the 2-level subdomain.
- Drizzle reads `DATABASE_URL` from `process.env` only — export it (or `dotenv -e`) when running `db:migrate`, else it falls back to the localhost default in `drizzle.config.ts`.
