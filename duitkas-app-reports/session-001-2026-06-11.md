**Key findings during analysis:**
- The API is minimal: it serves only `GET /health` + `/socket.io/*` (everything else 404s). The dashboard talks to the API purely over Socket.IO using `NEXT_PUBLIC_API_URL` (build-time baked). → single-subdomain path routing is trivially viable.
- The WhatsApp session is **not** at `~/data/sessions` as the env implies — `MIAW_SESSION_PATH=~/data/sessions` was never tilde-expanded, so the real session lives at `apps/api/~/data/sessions/catatkas-bot` (a directory literally named `~`, 829 files). It moves with the project folder, so keeping `MIAW_SESSION_PATH` + `MIAW_INSTANCE_ID` unchanged preserves the connection with **no QR re-scan**.
- There are **three** `.env` files (root + `apps/api/.env` + `apps/web/.env`). The API loads root `.env` first then `apps/api/.env` (**later overrides earlier**); `next build`/`next start` read `apps/web/.env`. All three had to be edited.
- The root `miaw-core` symlink (`../miaw-core` → nonexistent `/home/devops/miaw-core`) was **already dangling before this work** — unused; the real dependency is `node_modules/miaw-core@^1.1.1`.

## DNS / TLS blocker resolved

`baznas.duitkas.biji.uk` was orange-clouded through Cloudflare. Two problems:
1. CF intercepted port 80 with an "Always Use HTTPS" 301 at the edge → blocked certbot HTTP-01.
2. CF free Universal SSL covers `*.biji.uk` (one level) but **not** the 2-level `*.duitkas.biji.uk` → edge TLS handshake failed.

**Decision (user):** grey-cloud it (DNS-only) → A `46.250.234.153`, then issue a normal Let's Encrypt cert on the proxy. Simplest, native WebSockets, no CF cert limits. Must remain grey-clouded.

## Gotchas / notes

- **Keep `baznas.duitkas.biji.uk` grey-clouded.** Re-enabling Cloudflare proxy breaks TLS (free Universal SSL doesn't cover the 2-level subdomain).
- **Edit all three `.env` files** on any future config change — `apps/api/.env` overrides root for the API; `apps/web/.env` is what the web build/start read.
- `NEXT_PUBLIC_API_URL` is build-time baked — `rm -rf apps/web/.next` and rebuild web on any domain change.
- The WhatsApp session's literal-`~` path is a quirk of the unexpanded `MIAW_SESSION_PATH`; do **not** "fix" it to a real `~/data/sessions` without migrating the existing session dir, or it forces a fresh QR scan.
- Backups left on `app`: `~/miaw-flow-baznas/.env.bak-prebaznas`, `apps/{api,web}/.env.bak-prebaznas`, `ecosystem.config.cjs.bak-prebaznas`.
