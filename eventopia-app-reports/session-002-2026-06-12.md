## TLS — DONE (LE wildcard via DNS-01)

CF is **Full-strict**, so the proxy origin must present a valid `*.eventopia.my` cert. The existing
`cloudflare-gro.ini` token is zone-scoped to `gro.biz.id`, so a **new CF API token for the `eventopia.my`
zone** was supplied and stored at `/etc/letsencrypt/cloudflare-eventopia.ini` (chmod 600). Issued:

```bash
certbot certonly --dns-cloudflare \
  --dns-cloudflare-credentials /etc/letsencrypt/cloudflare-eventopia.ini \
  --dns-cloudflare-propagation-seconds 30 --cert-name eventopia.my \
  -d eventopia.my -d '*.eventopia.my' -m mdedirudianto@gmail.com --agree-tos --no-eff-email
```

cert `eventopia.my` (covers apex + wildcard), expires 2026-09-10, **auto-renew scheduled**. nginx blocks
repointed from the self-signed placeholder to `/etc/letsencrypt/live/eventopia.my/{fullchain,privkey}.pem`
and reloaded. **Verified through Cloudflare: all hosts 200** (526 cleared), apex→www, api `/readyz` ready,
landing renders, tenant 404 graceful.
