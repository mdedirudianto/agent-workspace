## What changed upstream

- `packages/whatsapp`: removed `verifyHandshake`/`subscribeWebhook`/`createTemplate`/`listRemoteTemplates`
  from the client; dropped `wabaId`/`verifyToken` from `WhatsAppConfig`.
- `apps/api`: removed `GET /webhooks/whatsapp` (the verify-token handshake route) — only `POST
  /webhooks/whatsapp` remains, now expecting payloads forwarded + re-signed by miaw-route.
- `packages/db`: migration `0071_wa_drop_verify_token.sql` drops the dead
  `organizer_whatsapp_config.verify_token` column (per-org `wabaId` column stays — admin/onboarding
  still read it).
- `.env.example`: dropped `META_WEBHOOK_VERIFY_TOKEN`/`META_WABA_ID`/`META_BUSINESS_ACCOUNT_ID`; several
  other vars clarified/renamed (`EVENTOPIA_ED25519_*` fully replaces old `JWT_SECRET`/`TICKET_TOKEN_*` —
  already true on both envs, nothing to do); object storage section re-labeled "PLANNED, not yet wired"
  (confirms session-006's finding unchanged); new `UPLOADS_BASE_URL`/`PUBLIC_UPLOADS_BASE` vars added
  (currently unset on both envs, defaulting to unroutable `https://uploads.local` — known gap, not fixed
  this session since object storage itself is still a stub).
- **No frontend app changed** (console/web-public/admin untouched) — redeploy was code pull + migration +
  `eventopia-api`/`eventopia-workers` restart only, no rebuild.
