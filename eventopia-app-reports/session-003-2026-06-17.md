# Session 3 — Eventopia public marketing face on `eventopia.co.id`

**Date:** 2026-06-17
**App:** `eventopia` (github.com/biji-dev/eventopia, branch `main`)
**Hosts:** `app` (`devops@154.26.129.104` / `10.0.0.5`) · ingress via `proxy` (`46.250.234.153` / `10.0.0.2`) · DB+Redis on `db` (`10.0.0.1`) — unchanged
**Domains:** **new** `eventopia.co.id` (Cloudflare orange) · existing `eventopia.my`
**Env:** `production`

Splits the public face from the organizer/attendee app across two TLDs:

- **`eventopia.co.id`** — public **marketing landing only** (apex + `www`).
- **`eventopia.my`** — the app: `app.` (console), `api.`, `<org>.` (tenant event/registration/checkout),
  `checkin.`. Apex + `www` now **301 → `eventopia.co.id`** (marketing moved off `.my`).

No new server, DB, or PM2 process — the **same `web-public` instance** (`10.0.0.5:38100`) serves both the
marketing root and the tenant pages; the Host header decides which.

## The one non-obvious gap

`web-public`'s `proxy.ts` middleware renders the marketing `Landing` only for the apex (`ROOT_DOMAIN`)
and reserved hosts; any other host is treated as a paid white-label custom domain → API resolve →
**404** if unknown. So pointing `eventopia.co.id` at the upstream **without a code change would 404** —
"just add a proxy vhost" was necessary but not sufficient. The marketing components are otherwise
self-contained (CTAs are WhatsApp/email/IG/anchors, no deep links into login/register), so no link
rewiring was needed.

## What was done

Commit **`3531450`** `feat(web-public): recognize a marketing-alias domain (eventopia.co.id)`:
- `apps/web-public/proxy.ts`: new env-driven `MARKETING_HOST` → its apex + `www` render the top-level

### 3. `proxy` — TLS + nginx
- **LE cert** `eventopia.co.id` (+ `www`) via **DNS-01** using the existing `.my` Cloudflare token
  (`/etc/letsencrypt/cloudflare-eventopia.ini` — confirmed it also authorizes the `eventopia.co.id`
  zone). Expires **2026-09-15**, auto-renew scheduled.
- **New isolated vhost** `/etc/nginx/sites-available/eventopia-coid.conf`: `:80`→`:443` redirect +
  apex/`www` 443 → `eventopia_web` (38100), **omitting `X-Forwarded-Proto`** (Next-standalone EPROTO
  fix, same as the `.my` web block). Reuses the `$eventopia_conn` map + `eventopia_web` upstream from
  `eventopia.conf`.
- **`eventopia.conf` edits** (backup `eventopia.conf.bak-20260617`): apex `eventopia.my` 301 target
  `www` → `eventopia.co.id`; trimmed `www` out of the `www + *.eventopia.my` web block (now
  `*.eventopia.my`); added an exact `www.eventopia.my` 443 block → 301 `eventopia.co.id`.
