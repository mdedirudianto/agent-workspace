### The seed gotcha worth remembering

The seed wrappers (`scripts/seed-organizer-showcase.sh`, the operator doc's one-liner) are
**Docker-lane scripts** ‚Äî they call `lane_resolve_db_url` to find a Docker PG container's host port
and **fail on bare-metal dev**. The underlying `.ts` honor inherited `DATABASE_URL` / `API_BASE_URL`,
so on dev we **bypass the wrapper** and run the `.ts` directly with explicit env (PG
`localhost:5432/eventopia`, API `127.0.0.1:38001`). PII keys stay on the in-code dev fallback (both
API and seed share the same fixed key ‚Üí encrypted KYC stays readable in the console).
