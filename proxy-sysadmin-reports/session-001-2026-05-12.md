## Findings

### Critical

| # | Finding | Detail |
|---|---------|--------|
| C1 | **Deprecated TLS versions enabled** | `nginx.conf` global SSL config includes `TLSv1 TLSv1.1` — both deprecated since 2020 (RFC 8996). Should be `TLSv1.2 TLSv1.3` only. Some individual site configs override this correctly, but the global setting is the fallback. |
| C2 | **cockpit.biji.uk is unintended default server** | No explicit `default_server` is set for port 443. Nginx falls through to the first server block (cockpit) for unmatched requests. Scanners hitting bare IP `46.250.234.153:443` land on cockpit config — generating constant `connect() failed (111)` errors because Cockpit service is down. |

### High

| # | Finding | Detail |
|---|---------|--------|
| H1 | **Cockpit service is down** | `cockpit.biji.uk` → `127.0.0.1:9090` is connection refused. Cockpit is not running on the proxy. nginx error log shows continuous scanner traffic hitting this dead backend. |
| H2 | **qute.dev backend is down** | `qute.dev` → `154.26.130.96:4321` is connection refused. External server (or port) is unreachable. nginx serving 502 to all qute.dev visitors. |
| H3 | **No swap configured** | 5.8 GB RAM with no swap. nginx peak memory usage was **2.0 GB** (shown in systemctl output). A memory spike could trigger OOM. |
| H4 | **api.wagrab.com missing `client_max_body_size`** | Error log shows `client intended to send too large body: 1255524 bytes` (1.2 MB) on `PATCH /items/bot_menus`. Directus uploads are being rejected. Other config files set `client_max_body_size 128M` (vault) or `40M` (errors) but wagrab-admin.conf has none. |

### Medium

| # | Finding | Detail |
|---|---------|--------|
| M1 | **fail2ban covers SSH only** | No nginx jail configured. Scanner traffic hitting the server is not rate-limited or banned at the firewall level. High-volume probe traffic is absorbed by nginx workers but generates noise in error logs. |
| M2 | **gro.biz.id uses `proxy_ssl_verify off`** | The ERP server (`10.0.0.6:443`) is proxied with SSL certificate verification disabled. If the internal cert is self-signed this is expected, but the connection between proxy and ERP is unverified. |
| M3 | **10.0.0.7 is reachable but unidentified** | Responds to ping, no SSH. Unknown node on the internal network. |
| M4 | **travelan-web.conf has no TLS** | Disabled config for `traveland.id` uses plain HTTP only — if ever re-enabled it would serve cleartext. Also points to external host `154.26.130.96:4001`. |
| M5 | **foucher-samarabatu.conf disabled despite cert existing** | `samara-batu.foucher.co` is covered by the `foucher.co` cert but the nginx config is not enabled. Cert renewal will still work, but the subdomain returns NXDOMAIN or unrouted. |
| M6 | **`ytgrab-web.conf` uses `$http_x_forwarded_proto` not `$scheme`** | `X-Forwarded-Proto` set to `$http_x_forwarded_proto` — if not set by upstream, will be empty. Should use `$scheme` for consistency with all other configs. |
| M7 | **`pasarbesar.id` main domain serves a dummy page** | The primary domain serves `/var/www/dummy` placeholder. Only `beta.pasarbesar.id` has a real backend. Visitors to the main domain get a 404 page. May be intentional (beta-only), but should be confirmed. |

### Informational

| # | Finding | Detail |
|---|---------|--------|
| I1 | **quranchat.me and qute.my share identical backends** | Both domains proxy to `10.0.0.5:3007` (web) and `:7005` (API). Same application, dual branding. The Nginx upstream label is `qute-api` / `qute-app` — these configs are effectively aliases. |
| I2 | **Cockpit nginx config has commented-out SSL cipher block** | `biji-cockpit.conf` has `ssl_protocols` and `ssl_ciphers` commented out. Relying on global nginx.conf settings (which include deprecated TLS — see C1). |
| I3 | **Several upstream blocks have commented-out fallback servers** | `pasarbesar`, `lpekinhub`, `teamy_app`, `teamy_web`, `wagrab-*`, `washop`, `ytgrab-*` all have `# server 10.0.0.3:xxxx;` or similar commented-out entries. These appear to be old monitor-server fallbacks from a previous topology. Dead weight in configs. |
| I4 | **nginx peak memory 2.0 GB (currently 37 MB)** | Systemctl shows `Memory: 37.4M (peak: 2.0G)`. The 2 GB peak was likely a traffic burst. Current state is healthy, but worth noting there's no memory limit on the nginx service unit. |
