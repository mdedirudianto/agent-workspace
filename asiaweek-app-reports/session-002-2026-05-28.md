## Gotchas Encountered

| Issue | Root cause | Fix |
|-------|-----------|-----|
| `NODE_ENV=development` crashed api + app | `apps/api` and `apps/app` both validate NODE_ENV with `z.enum(['local','staging','production'])` — 'development' not in enum | Changed to `NODE_ENV=local` in both `.env` and `ecosystem.config.cjs`; deleted+restarted all 3 processes (restart alone doesn't pick up new PM2 env) |
| PM2 restart didn't pick up new env | PM2 caches env from when the process was first started; `pm2 restart` reuses it | Must `pm2 delete` then `pm2 start ecosystem.config.cjs` to apply env changes |

## Redeploy Procedure (prod)

```bash
ssh devops@app "cd ~/asiaweek && git pull origin main && \
  set -a && . ./.env && set +a && \
  pnpm turbo run build && \
  pnpm --filter @asiaweek/db migrate && \
  pm2 delete asiaweek-web asiaweek-app asiaweek-api && \
  pm2 start ecosystem.config.cjs && \
  pm2 save"
```
