**Scope:** First deployment of the asiaweek monorepo to dev server (`dev.asiaweek.uk`); postgres DB, PM2 processes, nginx vhosts, SSL certs, migrations, seed.

## Context

asiaweek is a pnpm monorepo (Turborepo) with three apps:

| App | Framework | Purpose |
|-----|-----------|---------|
| `apps/web` | Astro 6 (static+SSR, Node adapter) | Public marketing site |
| `apps/app` | React Router v7 (full SSR) | Authenticated traveler/expert dashboard |
| `apps/api` | Hono + pg-boss | Backend HTTP API + async job runner |

Source lives at `~/Projects/asiaweek` on the local machine. Deployed to `deploy@dev` (`154.26.130.96`, Ubuntu 22.04) via rsync — not Docker, matching the dev server's existing PM2 pattern.

## Gotchas Encountered

| Issue | Root cause | Fix |
|-------|-----------|-----|
| `pnpm install` skipped esbuild/sharp/sentry-cli builds | pnpm 10 security lockdown requires explicit approval | Added `pnpm.onlyBuiltDependencies` to `package.json` on server |
| `asiaweek-api` crashed: `Cannot find package 'tsx'` | `tsx` is in `apps/api/node_modules` only; running from monorepo root doesn't resolve it | Changed PM2 `cwd` to `apps/api` so `node --import tsx` resolves locally |
| `asiaweek-app` crashed: shell syntax error in Node | PM2 tried to run the `.bin/react-router-serve` shell wrapper as Node | Changed to call `@react-router/serve/bin.js` directly via `node` |
| Cloudflare curl from server returned 000 | Curl going through Cloudflare DNS from origin itself causes loopback | Local `--resolve` test confirmed nginx/SSL is correct |

## Redeploy Procedure

```bash
# 1. Sync code from local
rsync -avz --delete \
  --exclude='node_modules' --exclude='.git' \
  --exclude='apps/web/dist' --exclude='apps/app/build' --exclude='.env' \
  ~/Projects/asiaweek/ deploy@dev:~/asiaweek/

# 2. On dev server: rebuild and restart
ssh deploy@dev "cd ~/asiaweek && set -a && . ./.env && set +a && \
  pnpm turbo run build && \
  pnpm --filter @asiaweek/db migrate && \
  pm2 restart ecosystem.config.cjs"
```
