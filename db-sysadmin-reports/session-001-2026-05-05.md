# Session 1 — Initial Audit & Hardening

**Date:** 2026-05-05
**Server:** root@db (`db.biji.uk`, public IP `109.123.233.215`)
**OS:** Ubuntu 22.04.5 LTS (Jammy Jellyfish), kernel 5.15.0-142-generic
**Purpose:** First-ever audit of this server — document what's running and fix the worst issues found.

## Actions Taken (this session)

1. **SSL cert** — renewed `db.biji.uk` Let's Encrypt cert via DNS-01/Cloudflare. New expiry 2026-08-03; auto-renewal active. Removed dead `meili.biji.uk` cert and the matching `/meili` Apache proxy.
2. **Origin firewall** — discovered ports 80/443 had no UFW rules (origin was unreachable from public internet via Cloudflare). Added 20 UFW rules permitting only Cloudflare's published IPv4/IPv6 ranges. Direct-to-IP origin access remains blocked.
3. **Netdata exposure** — port 19999 was bound to `0.0.0.0` with no UFW rule. Added ALLOW from `10.0.0.0/22` and DENY from anywhere else.
4. **Mongo Express** — was running as root. Migrated to dedicated `mongo-express` system user, app moved from `/root/mongo-express` to `/opt/mongo-express`.
5. **Swap** — added 4 GB swapfile at `/swapfile`, persisted in `/etc/fstab`.
6. **MariaDB credentials** — located root password (shared with `phpmyadmin` user, value `mypr0d123`), saved to `/root/.my.cnf` (mode 600). Enumerated databases (one ERPNext site at 47 MB).
7. **PgBouncer** — was running but completely unconfigured (empty `[databases]`, empty `userlist.txt`, zero connections). Stopped and disabled.
8. **Default Apache homepage** — replaced the Ubuntu "It works!" page with a minimal blank page; no OS/version branding leaked anymore.
9. **PostgreSQL public listener removed** — was binding to `127.0.0.1`, `10.0.0.1`, and `109.123.233.215` (public). Changed `listen_addresses` to `localhost,10.0.0.1` only and restarted. UFW already blocked 5432 from non-LAN sources, so this is defence-in-depth: even if UFW is misconfigured/disabled, PostgreSQL is no longer reachable on the public IP.
10. **MariaDB orphan user dropped** — `_2ddb56be586f473b@erp` (leftover from a deleted ERPNext site, only `USAGE` privilege, no matching DB) removed.
11. **MariaDB password rotation** — split the shared `mypr0d123` password. `phpmyadmin@localhost` got its own new strong password (config-db.php and dbconfig-common/phpmyadmin.conf updated in lockstep). All 6 `root@*` entries (`localhost`, `10.0.0.1`, `10.0.0.2`, `10.0.0.3`, `10.0.0.4`, `backup`) rotated to a single new strong password; `/root/.my.cnf` updated. New passwords recorded in `/root/.mariadb-rotation-2026-05-06.txt` (mode 600). Backups of the previous configs left at `*.bak.2026-05-06`.

Cloudflare API token used for DNS-01 stored at `/root/.cloudflare.ini` (mode 600).

## Origin / Cloudflare topology

`db.biji.uk` is fronted by Cloudflare (orange-cloud / proxied). TLS to clients is terminated by Cloudflare with its `*.biji.uk` certificate; Cloudflare connects back to origin (`109.123.233.215`) on port 443 using the Let's Encrypt cert installed on Apache.

Origin firewall is locked down to Cloudflare's published IPv4/IPv6 ranges (20 CIDRs, fetched from `https://www.cloudflare.com/ips-v4` and `/ips-v6`). Direct origin access bypassing Cloudflare is blocked. Cloudflare occasionally updates these ranges — refresh periodically with:

```bash
for ip in $(curl -s https://www.cloudflare.com/ips-v4) $(curl -s https://www.cloudflare.com/ips-v6); do
    ufw allow proto tcp from $ip to any port 80,443 comment 'cloudflare-edge'
done
ufw reload
```

(UFW silently skips duplicates; old rules for IPs no longer in Cloudflare's list need manual cleanup.)
