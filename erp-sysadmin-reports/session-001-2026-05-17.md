## Upstream Topology — `proxy` → `erp`

Confirmed by reading `proxy:/etc/nginx/sites-enabled/gro.biz.id.conf`:

```
Internet ──► proxy (46.250.234.153)         ──► erp (10.0.0.6)
              :80  → 301 https              :443 (HTTPS, internal)
              :443 (wildcard *.gro.biz.id)    └─ nginx → gunicorn :8000
                ├─ /          ──proxy_pass─►       └─ /socket.io ──► socketio :9000
                └─ /socket.io ──proxy_pass─►
```

Key details from the proxy config:
- The wildcard SSL cert lives on **both** boxes (`/etc/letsencrypt/live/gro.biz.id/`) and proxy→erp uses `https://` with `proxy_ssl_verify off`. Renewals need to stay in sync across both hosts.
- `proxy_buffer_size 128k; proxy_buffers 4 256k;` — tuned for ERPNext's chunky responses.
- `proxy_*_timeout 300s` — long timeouts for slow report generation.
- Future `grobiz` app on `app` will be able to call ERPNext via the **internal** address `https://10.0.0.6` (no Internet round-trip).

## Security Findings

### Critical

| # | Finding | Detail |
|---|---------|--------|
| C1 | **`PermitRootLogin yes`** | Direct root SSH allowed. `PasswordAuthentication` is contradicted across configs (`50-cloud-init.conf: yes`, `60-cloudimg-settings.conf: no`) — last file wins so password auth is effectively off, but the configuration is confusing and brittle. |
| C2 | **No fail2ban, no UFW — 7,292 failed SSH auth events in current `auth.log`** | Top offenders: `45.156.87.69` (1,590 attempts), `138.197.163.192` (777), `72.24.235.63` (306), `168.222.142.246` (221). No rate-limit, no ban list. |
| C3 | **5 kernels behind + 64 packages pending — `/var/run/reboot-required` set** | Running `6.8.0-100`; installed but not booted: `-101, -106, -107, -110, -111`; apt has `linux-image-virtual 6.8.0-117` queued. Cluster of CVEs since Jan 2026 not picked up. |
| C4 | **Netdata exposed on `0.0.0.0:19999` without auth** | Full system metrics, process list, configuration visible to anyone who finds the IP. Same exposure pattern that was fixed on `app` in session-001. |

### High

| # | Finding | Detail |
|---|---------|--------|
| H1 | **node socketio bound to `*:9000` (public)** | Frappe socketio is meant to be proxied through nginx (`/socket.io/`). Direct bind on the public interface bypasses nginx, leaks the realtime port, and is exploitable for protocol-level abuse. |
| H2 | **No swap on 11 GB box running 20 gunicorn workers** | Current memory use is fine (2.3 GB / 11 GB), but `--preload` + 20 workers + a large ERPNext request set could spike past available RAM with no fallback. App server hit the same gap and added 2 GB swap in session-001. |
| H3 | **Old RSA key `rsa-key-20160704` in root authorized_keys** | Same key flagged on `app` in session-001 — almost certainly a forgotten 10-year-old credential. |
| H4 | **No host firewall at all (`ufw inactive`)** | All other servers (`numa`, `app`) have an active UFW with explicit rules. `erp` has nothing — every listening port is implicitly public. |

### Medium

| # | Finding | Detail |
|---|---------|--------|
| M1 | **SSL wildcard cert expires 2026-07-24 (~9 weeks out) — *and* lives on two hosts** | The same `gro.biz.id` wildcard is deployed on both `proxy` and `erp`. No `certbot.timer` was visible on `erp` in `systemctl list-timers`. If renewal is automated only on `proxy` (likely, since that's where it terminates publicly), the copy on `erp` will go stale — and proxy→erp HTTPS will start failing the moment `proxy_ssl_verify` is ever turned on. Document which host is the source of truth and how the cert reaches the other. |
| M2 | **Backups not verified** | `/home/frappe/frappe-bench/sites/erp.gro.biz.id/private/backups/` not inspected for recency. ERPNext business data — needs a known-good off-host backup. |
| M3 | **`frappe_user: frappe` + supervisor running as root** | Standard frappe-bench install, but worth documenting: any compromise of the `frappe` user gets full app + DB-credential access via `site_config.json`. |
| M4 | **2 failed systemd units** | `cloud-init.service` and `systemd-networkd-wait-online.service` — boot-time, not currently impactful. Cosmetic. |
