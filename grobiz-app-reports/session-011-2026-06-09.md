## Notes / follow-ups
- Reveal is **per-tenant only**; on first reveal it rotates the tenant's ERPNext Administrator password via `bench set-admin-password` over the existing provisioning SSH key and stores the AES-256-GCM ciphertext in `tenants.erpnext_admin_password_enc`. End-to-end reveal via the admin UI **not exercised this session** (no live admin session driven) — code path + encryption verified, SSH runner configured. On **dev** the reveal is inert (shared mode).
- `ERPNEXT_CRED_ENC_KEY` is **load-bearing for data, not just access**: once a tenant's password is encrypted with it, rotating/losing the key makes existing ciphertext undecryptable (next reveal just rotates again, so non-fatal, but note it for `.env`/backup hygiene). Prod + dev now have **distinct** keys (independently generated).
