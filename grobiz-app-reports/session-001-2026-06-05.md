## Context

`grobiz` (`github.com/biji-dev/grobiz`) is a multi-tenant SaaS for Indonesian UMKM (POS / loyalty / WhatsApp), built as a **pnpm monorepo** ‚Äî *not* Next.js. Each business tenant gets its **own ERPNext site** (`<8hex>.gro.biz.id`) auto-created on signup; grobiz is the middleware in front.

Deployable surface:

| App (`apps/*`) | Type | Serves |
|---|---|---|
| `api` | Fastify 5 (run with `node --import tsx/esm`) | REST API + DB-polled provisioning worker + 11 cron schedulers; **also serves the `web` SPA** via `@fastify/static` |
| `web` | React 19 + Vite PWA | product app |
| `admin` | React 19 + Vite SPA | internal admin dashboard |
| `landing` | static HTML | marketing site |
| `mobile`, `extension` | RN / Chrome ext | not server-deployed |

Backends: **PostgreSQL + Redis on `db` (10.0.0.1)**, **ERPNext on `erp` (10.0.0.6)**. Cluster pattern: PM2 on `app`, `proxy` nginx reverse-proxies per subdomain.

## Gotchas captured

- **`bench` over non-interactive SSH** fails from `$HOME` (not a bench dir) and only the `/usr/local/sbin/bench` shim (cd then exec) makes grobiz's bare `ssh frappe@erp "bench ..."` work. PATH was fine; cwd was the problem.
- **`root@10.0.0.6` was missing** on db MariaDB (root allowed from .1/.2/.3/.4/backup but not erp). `bench new-site --mariadb-root-password` authenticates as `root` ‚Üí had to add it.
- **`common_site_config` had no `db_host`** ‚Üí new sites would default to localhost (no MariaDB on erp). Set `db_host=10.0.0.1`.
- **Redis db3 already had 2103 keys** ‚Äî verify with `redis-cli -n N dbsize` before assigning; grobiz uses **db4**.
- erp's bench nginx was already wildcard host-routing (`server_name *.gro.biz.id`, `X-Frappe-Site-Name $host`) ‚Üí **no nginx regen needed** for new tenant sites.
