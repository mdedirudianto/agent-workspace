## The tax provisioning, in code
New `apps/api/src/core/erpnext/templates/tax-setup.ts` (commit `68d9584`), wired into `provisionCompany()` as **step 12b**, *after* `cleanupDefaultAccounts` (which deletes every Sales Taxes and Charges Template for the company — so the seeded template must come after it). `provisionTaxConfig()` does two idempotent things per company:
1. `updateDoc('Account', '2130 - Utang Pajak - <abbr>', { account_type: 'Tax' })` — so ERPNext treats PB1/PPN output-tax rows correctly (reuses the existing SAK EMKM liability account `2130` rather than nesting `2131/2132` leaves, keeping `2130` a postable leaf). Service charge books to income `4200 - Pendapatan Lain-lain`.
2. `createDocIfNotExists('Sales Taxes and Charges Template', 'Pajak <Company>')` seeded with the business-type default row.

A tenant uses one tax type at a time (PB1 *or* PPN), so one shared payable account suffices; the per-row `description` distinguishes them. This step only runs for **new** provisioning — hence existing tenants needed the backfill below.

## Tax-account backfill on existing tenants (the "remaining op")
The stock `pnpm erpnext:init` only re-provisions the single fixed **dev convenience** tenant (`dev00001` / "Test Company") **and rotates its API keys** as a side effect — it would cover only 1 of 8 companies. So instead I ran a **targeted, minimal-blast-radius backfill** that calls only `provisionTaxConfig()` (the exact step-12b logic) for **every** company on the shared site:
- One-off script `apps/api/src/scripts/backfill-tax-accounts.ts` (modeled on `erpnext-init.ts` env-loading), run via `pnpm --filter @grobiz/api exec tsx` (resolves the `@grobiz/shared` workspace import). It fetched the 8 companies from ERPNext, mapped `business_type` per company from the `tenants` table (default `retail`), called `provisionTaxConfig(erpnext, company, abbr, type)`, then read back each `2130.account_type`.
- **Result: 8/8 → `account_type='Tax'`.** Script deleted afterward (not committed); working tree clean (only `.env` backups untracked).
