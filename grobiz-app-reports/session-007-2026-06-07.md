## api.dev.gro.biz.id setup (dev live pricing)
The dev landing fetches `https://api.dev.gro.biz.id/api/public/plans`, but that host didn't exist and the dev `plans` table was empty. Fixed:
1. **DNS** (user, Cloudflare `gro.biz.id` zone): added `A api.dev → 154.26.130.96`, **DNS-only (grey cloud)** — matches the other dev records; required so Let's Encrypt HTTP-01 can validate.
2. **Seed plans**: `pnpm seed:defaults` on dev (idempotent) → 4 plans inserted (free/starter/growth/max, all `is_active`), +5 flags/3 config/6 templates. Reads `.env` via dotenv (cwd).
3. **Bust stale cache**: `/api/public/plans` is **Redis-backed** (`CacheService`, key `plans:public:list`, 1h TTL, no prefix) — the empty `[]` cached during first verify survived a process restart. `redis-cli -h localhost -n 0 DEL plans:public:list` → next request repopulated from the seeded rows. (A PM2 restart does NOT clear this; delete the Redis key or edit a plan in admin to bust.)
4. **nginx vhost** `/etc/nginx/sites-available/api.dev.gro.biz.id.conf` (proxy `/` → `127.0.0.1:7006`), symlinked into sites-enabled.
5. **TLS**: `certbot --nginx -d api.dev.gro.biz.id` → cert issued (expires 2026-09-05), 443 block + HTTP→HTTPS redirect added, auto-renew scheduled.
