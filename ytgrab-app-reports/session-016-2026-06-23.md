## Summary

Routine full-stack redeploy bringing `app` from web/api **1.1.3** to **1.2.0** — the
**"proxy gate overhaul"** branch. The previous gate (session-015) disabled a proxy on
any failure, which had churned the pool down to **20 active datacenter / 0 active
residential** (70 disabled DC, 70 recycled DC, 8 disabled res). 1.2.0 replaces that with a
less-aggressive **strike model**: only *proxy-fault* failures count toward the gate,
disabled proxies get an `archived` terminal state, and a new admin
**`POST /admin/proxies:reset`** endpoint/button can revive the pool on demand.
