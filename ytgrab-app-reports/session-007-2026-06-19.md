### The IPv4 egress fix (`529f454`, merge `a22021e`)
Pins all YouTube egress to IPv4. googlevideo stream URLs are IP-locked: yt-dlp may resolve metadata over IPv6 while Node/undici's Happy Eyeballs picks IPv4 for the byte fetch → IP mismatch → `302`/`403`/`ECONNRESET` on download. Fix forces IPv4 on **both** yt-dlp (`--force-ipv4`) **and** undici (dns `ipv4first` + `autoSelectFamily` off), gated by env `FORCE_IPV4` (default `true`).

| File | Change |
|---|---|
| `apps/api/src/env.ts` | `FORCE_IPV4` env (`z`, default `true`) |
| `apps/api/src/index.ts` | undici global dispatcher → IPv4-first, autoSelectFamily off |
| `apps/api/src/extractors/{ytdlp,ytdlp-streamurl,captions}.ts` | `--force-ipv4` on yt-dlp |
| `.env.example` | documents `FORCE_IPV4` |
| `apps/api/tests/extractors/force-ipv4.test.ts` | coverage (+71 lines) |

**`apps/api/.env` already had `FORCE_IPV4=true`** — no env change needed.
