# Session 6 — prom-client route-label memory leak fix + API-only redeploy to `1017f37`

**Date:** 2026-06-17
**Scope:** Root-caused and fixed a steady `ytgrab-api` memory leak (~200 MB/day → 2.5 GB over 12 days) that exhausted RAM on the `app` server, drove it into swap-thrash, and surfaced as a **`routes.biji.uk` 504** (miaw-route froze while paged out). Fix deployed to **`app` API only**; `ytgrab-web` deliberately left on its current build.

> Cross-ref: host-side diagnosis, fleet swap/Telegram alerting, and PM2 cap rollout are in [app-sysadmin-reports/session-004](../app-sysadmin-reports/session-004-2026-06-17.md).

---

## Root cause

`apps/api/src/middleware/logging.ts`:

```ts
const route = c.req.routePath ?? c.req.path;   // BUG
httpRequestsTotal.labels(method, route, status).inc();
httpRequestDurationSeconds.labels(method, route, status).observe(durationSec);
```

`c.req.routePath` is the **templated** route (`/api/v1/videos/:id`) — bounded. But on **unmatched routes (404s)** `routePath` is `undefined`, so it falls back to `c.req.path` — the **raw, caller-controlled URL**. On the public API (`api.youtubegrab.com`) bots probe thousands of unique paths (`/.env`, `/wp-login.php`, random IDs), and **prom-client keeps every unique label combo as a permanent time series** (the Histogram especially: buckets + `_sum` + `_count` per combo). Unbounded heap growth, linear with unique probe URLs → ~200 MB/day.

## Fix

```ts
const route = c.req.routePath ?? "unmatched";
```

One line. Bounds cardinality to the set of defined routes plus a single `"unmatched"` bucket.

### Scope control — the 14-commit gap (how the hotfix stayed isolated first)
The `app` checkout was **14 commits behind `origin/main`** at the time of the fix: **unreleased product work** (new `apps/web` `/install` pages, extension-pill rework, `env.ts`; the `apps/mobile` MVP; a dev-port renumbering chore; a ~5,600-line `pnpm-lock.yaml` dependency bump) — **none touched `apps/api/src`**. The hotfix was deliberately isolated first: fix branch cut from `origin/main` via a `git worktree`, API built from the existing checkout, merged `--no-ff` to `main`, server `reset --hard` to `1017f37` (API-only redeploy). **Then, on user approval, web was rebuilt from `main` and shipped** — so the 14 commits are now live. No more split state; `app` api+web both on `1017f37`.
