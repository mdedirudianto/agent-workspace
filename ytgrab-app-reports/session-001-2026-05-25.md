## Context

ytgrab is a revamped TypeScript monorepo (pnpm + Turborepo) replacing the legacy Koa+Vue SSR stack. The production stack is Docker Compose (`infra/docker/docker-compose.yml`) with services: `postgres`, `redis`, `migrate`, `api`, `web`, `nginx`, `certbot`, plus optional `vector` (observability) and `backup` (S3) profiles.

The `infra/docker/.env` file previously contained dev placeholder values (`postgres123`, `dev-secret-change-me`, `localhost` URLs). This session replaces them with production-ready values.

## Pre-Deploy Checklist

Before running `docker compose up -d` on app server:

- [ ] Set `YTGRAB_IMAGE_TAG` to the actual release tag (not `latest`)
- [ ] Set `ADMIN_EMAILS` to seed the first admin user
- [ ] Set `NEXT_PUBLIC_OPENPANEL_CLIENT_ID` (or leave blank to skip analytics)
- [ ] Ensure proxy nginx config covers `youtubegrab.com` + `api.youtubegrab.com` → app server ports 80/443

- [ ] After first `up -d`, run `pnpm db:migrate` (or confirm migrate service ran successfully)
- [ ] Hit `https://youtubegrab.com` — verify web loads
- [ ] Hit `https://api.youtubegrab.com/api/v1/health` — verify API responds
