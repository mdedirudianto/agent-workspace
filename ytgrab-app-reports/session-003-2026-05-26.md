### Gotcha: turbo doesn't track `apps/web/.env`

`turbo.json` has no `env:` block on the `build` task, so `NEXT_PUBLIC_*` changes in `apps/web/.env` do **not** invalidate the turbo cache. First build on app server hit `FULL TURBO` and restored a `.next/` directory **without** `standalone/` (the prior cached build was a pre-standalone artifact), leaving pm2 unable to find `server.js`.

**Workaround used:** `rm -rf apps/web/.next` before `pnpm --filter @ytgrab/web build`.

**Permanent fix (deferred):** Add to `turbo.json`:

```json
"build": {
  "dependsOn": ["^build"],
  "env": ["NEXT_PUBLIC_*"],
  "outputs": ["dist/**", ".next/**", "!.next/cache/**"]
}
```

## Part B — Webshare rotating proxy on prod (app server only)

Goal: bypass YouTube's "Sign in to confirm you're not a bot" challenge that fires on datacenter IPs.

### Initial attempt (failed)

```
YTDLP_PROXY=http://mbmkuoqx:i9h6jrarzm6e@p.webshare.io:80
```

Result: `407 Proxy Authentication Required — "The proxy you are connecting is not in your list."`

### Fix: `-rotate` username suffix

```
YTDLP_PROXY=http://mbmkuoqx-rotate:i9h6jrarzm6e@p.webshare.io:80
```

Direct curl test from app server: `curl -x http://...@p.webshare.io:80 https://api.ipify.org` → `64.137.14.176` (rotating exit IP, not app server's IP `154.26.129.104`) ✓

```bash
curl -X POST https://youtubegrab.com/api/v1/videos:process \
  -H "Content-Type: application/json" \
  -d '{"url":"https://www.youtube.com/watch?v=dQw4w9WgXcQ"}'
```

Returns full `VideoDetails` for Rick Astley video — no bot challenge ✓
