**Goal:** Install Fluent Bit and ship all PM2 process logs on `dev` to a dedicated `dev` stream in OpenObserve (`observe.biji.uk`), keeping it cleanly separated from the production `app` stream.

---

## Design Decision: Dedicated `dev` Stream

`dev` runs many of the same app names as the production `app` server (`feyfa-wa`, `wagrab-admin`, `pasarbesar`, `ytgrab-be`, etc.). Rather than mixing dev and prod logs in the same `app` stream, a separate `dev` stream was created:

| Concern | Approach |
|---------|----------|
| Log mixing | Separate stream `dev` — prod dashboards query `FROM "app"` exclusively |
| Retention | `dev` stream set to **7 days** vs global 3650-day default |
| Transport | HTTPS to `observe.biji.uk:443` (dev is not on `10.0.0.x` LAN) |
| Tagging | `env=dev`, `server=dev` added to all records |

## Fluent Bit Config

**`/etc/fluent-bit/fluent-bit.conf`:**

```ini
[SERVICE]
    flush        5
    daemon       Off
    log_level    info
    parsers_file parsers.conf
    plugins_file plugins.conf

[INPUT]
    name              tail
    tag               pm2.logs
    path              /home/deploy/.pm2/logs/*-out.log,/home/deploy/.pm2/logs/*-error.log
    path_key          source
    db                /var/lib/fluent-bit/pm2-logs.db
    mem_buf_limit     10MB
    skip_long_lines   On
    refresh_interval  10

[FILTER]
    name   lua
    match  pm2.logs
    script /etc/fluent-bit/pm2-extract.lua
    call   extract_app

[FILTER]
    name   record_modifier
    match  pm2.logs
    record server dev
    record host   dev
    record env    dev

[OUTPUT]
    name        http
    match       pm2.logs
    host        observe.biji.uk
    port        443
    uri         /api/default/dev/_json
    format      json
    http_user   admin@biji.uk
    http_passwd <redacted>
    tls         on
    tls.verify  on
    compress    gzip
```

## Not Covered (gap)

The Docker `grobiz-*` containers (erpnext, worker, scheduler, socketio, mariadb, redis, nginx) log to Docker's JSON log driver. The current Fluent Bit config only tails PM2 logs under `/home/deploy/.pm2/logs/` — Docker container logs are **not** shipped. A separate `tail` input pointing at `/var/lib/docker/containers/.../*.log` (with Docker JSON parser) would be needed to cover them.
