# Session 1 — Migrate Meta connection from direct tunnel → miaw-route

**Date:** 2026-06-10
**Scope:** Stop pointing Meta's BIJI Dev webhook directly at wabacall's cloudflared quick-tunnel. Give wabacall a stable public ingress (`wabacall.dev.biji.uk`) and route its call webhooks through the shared production `miaw-route` router instead.

---

## Background — what "direct" meant

wabacall runs on `dev` (`deploy` user, PM2 `wabacall`, `node server.js` on `0.0.0.0:19000`). Its webhook handler is `GET/POST /call-events`.

It was exposed to Meta via a cloudflared **quick tunnel** (PM2 `wabacall-tunnel` → ephemeral `*.trycloudflare.com`). The script [`scripts/start-tunnel.sh --update-meta`](../) did the actual wiring — it `POST`ed to the Graph API to set the **app-level** subscription:

```
POST graph.facebook.com/v24.0/1125824895132994/subscriptions
  object=whatsapp_business_account
  fields=calls,messages
  callback_url={trycloudflare-url}/call-events
  verify_token={wabacall VERIFY_TOKEN}
```

Because Meta allows only **one** webhook URL per app, this overwrote the entire **BIJI Dev** app subscription (shared with PasarBesar / GroBiz) and pointed it at a throwaway tunnel → wabacall. That is the "direct" coupling this session removes.

### Route added (prod `miaw_route` DB on `10.0.0.1`)

```sql
INSERT INTO routes (name, phone_number_id, waba_id, backend_url, app_id, enabled, fan_out, is_default, priority)
VALUES ('WabaCall - DEV', '109689798856637', '100405506465612',
        'https://wabacall.dev.biji.uk/call-events',
        'd7745f14-1e19-4836-8ca1-866ed58cb4d6',  -- BIJI Dev app
        true, false, false, 0);
```

Single **primary** route (`fan_out=false`): wabacall receives both `calls` and `messages` for the number and ignores the messages — confirmed acceptable (no prod messages consumer for this number).

### nginx vhost (`/etc/nginx/sites-available/wabacall.dev.biji.uk.conf`)

80→443 redirect + 443 TLS (wildcard `*.dev.biji.uk` cert) → `proxy_pass http://localhost:19000` with WebSocket upgrade headers (`Upgrade`/`Connection $connection_upgrade`) for the operator UI. The Meta webhook (`POST /call-events`) is plain HTTP and works regardless.

## Gotchas / notes

- **`start-tunnel.sh --update-meta` is hard-disabled.** Re-running it would clobber the shared BIJI Dev subscription back to a throwaway tunnel, breaking miaw-route routing for wabacall **and** PasarBesar/GroBiz on the same app. Re-enable only to deliberately revert to the direct model.
- The `$connection_upgrade` map did not previously exist on `dev` — added as a `conf.d` snippet (global), reusable by future WS vhosts.
- Prod miaw-route runs under **systemd**, not PM2/Docker — restart via `systemctl restart miaw-route`, logs `journalctl -u miaw-route -f`. (The MEMORY index line previously said "PM2"; corrected this session.)
- The local guard edit (`~/Playground/wabacall/scripts/start-tunnel.sh`) is an **uncommitted** working-tree change; the dev copy was patched in place. Commit + redeploy when ready.
- True end-to-end verification (a real call landing in prod `webhook_logs` and reaching wabacall) was left to the user / miaw-route side per request — this session stopped at "route present + ingress reachable".
