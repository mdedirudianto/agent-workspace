## Actions Taken

| # | Action | Detail | Result |
|---|--------|--------|--------|
| 1 | **Confirmed LAN connectivity** | `app` has internal IP `10.0.0.5/22` on `eth1`. `curl http://10.0.0.3:5080/` → 308. Logs ship over LAN directly — no public internet round-trip. | Done |
| 2 | **Identified PM2 log paths** | All 17 app logs in `/home/devops/.pm2/logs/`. Pattern: `<app>-out.log` (stdout) + `<app>-error.log` (stderr). Rotated files use `<app>-out__YYYY-MM-DD_00-00-00.log` format — excluded by glob. | Done |
| 3 | **Installed Fluent Bit v5.0.6** | Official apt repo (`packages.fluentbit.io/ubuntu/noble`), same as proxy. Binary at `/opt/fluent-bit/bin/fluent-bit`, systemd unit at `/lib/systemd/system/fluent-bit.service`. | Done |
| 4 | **Wrote `/etc/fluent-bit/pm2-extract.lua`** | Lua filter: extracts `app_name` and `log_type` (`stdout`/`stderr`) from the `source` filename field. e.g. `/home/devops/.pm2/logs/feyfa-wa-out.log` → `app_name=feyfa-wa`, `log_type=stdout`. | Done |
| 5 | **Wrote `/etc/fluent-bit/fluent-bit.conf`** | Single `tail` input with glob `*-out.log,*-error.log` + `path_key source`. Lua filter for app name. `record_modifier` adds `server=app`, `host=app.biji.uk`. HTTP output → `http://10.0.0.3:5080/api/default/app/_json`. | Done |
| 6 | **Enabled & started service** | `systemctl enable --now fluent-bit`. All 34 log files (17 apps × 2) watched via inotify. | Active (running) |
| 7 | **Verified** | `HTTP status=200`, `successful:21, failed:0` within 8s of start. OpenObserve query confirms 9 apps reporting in first minute with correct `app_name` + `log_type`. | Done |
