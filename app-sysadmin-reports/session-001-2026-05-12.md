## Findings

### Critical

| # | Finding | Detail |
|---|---------|--------|
| C1 | **PM2 logs consuming 5.1 GB** | `wagrab-admin-out.log` = **4.7 GB** alone. No log rotation configured. Will fill the 100 GB disk over time. |
| C2 | **Active SSH brute force, no fail2ban** | Attacker `118.37.92.184` hitting port 22 continuously with common usernames (`admin`, `oracle`, `ubuntu`, `test`, etc.). No rate limiting or banning in place. |
| C3 | **`PermitRootLogin yes`** | Direct root SSH access allowed. Combined with C2, this is a significant exposure. |
| C4 | **`miaw-api` + `miaw-web` bound to `0.0.0.0`** | Ports 7007 and 7008 are publicly accessible without any nginx/proxy layer. Direct exposure of application ports. |

### High

| # | Finding | Detail |
|---|---------|--------|
| H1 | **`miaw-web` has 262 restarts** | Near-constant crash loop over 19 days. App is recovering but something is wrong. |
| H2 | **Netdata port 19999 is publicly accessible** | Monitoring dashboard (metrics, process list, system info) exposed without auth. |
| H3 | **No swap configured** | 11 GB RAM with no swap. `ytgrab-fe` alone uses 1.3 GB. A memory spike could trigger OOM kills. |
| H4 | **Old 2016 RSA key in root authorized_keys** | `rsa-key-20160704` should be audited — likely a legacy key from a different era. |

### Medium

| # | Finding | Detail |
|---|---------|--------|
| M1 | **`pasarbesar-lpekinhub` has 25 restarts, `pasarbesar` has 17** | Both Next.js apps restarting frequently. May indicate memory leaks or startup errors. |
| M2 | **No log rotation on build logs** | `feyfa-voucher-rebuild.log` = 42 MB. Build logs accumulating in `/home/devops/logs/`. |
| M3 | **nginx installed but unused** | nginx is running the default stub page — not routing any traffic. Either a dead remnant or never finished setup. |
| M4 | **Docker running with zero containers** | Docker daemon is active with no workload. Small idle overhead; may be legacy from earlier setup. |
| M5 | **No git remotes on 17/18 repos** | Repos are local-only. No CI/CD pull path — deploys require manual push to server. Recovery from disk failure would require local backups. |
