## DB migration — `timestamp` → `timestamptz` (the careful part)

7 columns converted: `apps.{created_at,updated_at}`, `routes.{created_at,updated_at}`, `webhook_logs.{received_at,forwarded_at}`, `settings.updated_at`.

### Why a blind `drizzle-kit push` was unsafe
`ALTER COLUMN ... TYPE timestamptz` **without a `USING` clause** interprets existing naive values in the *session timezone*. Get that wrong and every timestamp silently shifts.

### Investigation (db `10.0.0.1`, database `miaw_route`)
- Postgres session `TimeZone` = **Asia/Singapore (+08)** (server OS is Asia/Jakarta, but PG default differs).
- App driver = `drizzle-orm/bun-sql` (`new SQL(DATABASE_URL)`) — **no TZ override**, inherits server default.
- Latest `received_at` = `2026-06-06 16:19` (naive) vs real time `17:04+08` / `09:04 UTC` → values are clearly **Singapore-local (+08)**, not UTC.

### Migration done (explicit, instant-preserving)
1. **Backup**: `pg_dump miaw_route` → `/root/miaw_route-backup-20260606-160602.sql` (16K).
2. Transactional ALTER ×7 with `USING col AT TIME ZONE 'Asia/Singapore'` — interprets the naive values as Singapore, stores the correct UTC instant.
3. **Verify**: `received_at` `16:19:26.344658` → `16:19:26.344658+08` (same wall-clock instant ✅); defaults intact (`received_at`→`now()`).
4. **Confirm sync**: after pulling the new schema, `drizzle-kit push` → **"No changes detected"** ✅.

Tiny data (apps=1, routes=3, settings=5, webhook_logs=10) → no lock concerns.

## Deploy procedure (on `app`, ordering mattered)

DB migrated **before** code pull (manual ALTER), then:
1. Rollback point `773d064` → `~/miaw-route-rollback-commit.txt`
2. `git pull --ff-only origin main` → `89c5101`
3. `bun install --frozen-lockfile` — no dep changes
4. `bun run css:build` — **CSS regenerated 84563→85260** (new reconciler settings/app-form classes); committed back in `bb45dce`
5. `db:push` (with explicit `DATABASE_URL`) → "No changes detected"
6. `pm2 restart miaw-route --update-env && pm2 save`

> ⚠️ `bun run db:push` does NOT load `.env.production` — drizzle-kit needs `DATABASE_URL` passed explicitly: `DATABASE_URL="$(grep ^DATABASE_URL= .env.production | cut -d= -f2-)" bun run db:push`. Don't `source` the env file (other values contain spaces).
