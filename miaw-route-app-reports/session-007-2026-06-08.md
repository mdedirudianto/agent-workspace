# Session 7 — Standardize all backends on internal IP `10.0.0.5` + grobiz off public NIC

**Date:** 2026-06-08
**App:** miaw-route v0.9.7 (no redeploy) · grobiz-api v0.9.4 (rebind + 1 code fix)
**URL:** https://routes.biji.uk
**Scope:** routing-address standardization (miaw-route DB) + grobiz bind hardening

---

## Goal

After sessions 005–006 the three routes used an **inconsistent** convention —
GroBiz `localhost:4000`, PasarBesar `10.0.0.5:7002`, Default Echo `localhost:9001`. The
split existed only because of how each app **binds**: grobiz-api on `0.0.0.0` (loopback
works), pasarbesar on `10.0.0.5` only. Standardize everything on the internal cluster IP
**`10.0.0.5`** — one convention, and it takes grobiz **off the public NIC** (defense in
depth: UFW already blocks `:4000` publicly, but `0.0.0.0` still listened on the public
interface).

`127.0.0.1` was rejected: the cluster `proxy` (10.0.0.2) reaches both apps for public
ingress at `10.0.0.5:PORT` (`api.gro.biz.id → proxy_pass http://10.0.0.5:4000`), so they
must listen on the internal IP, not loopback-only.

> miaw-route itself stays bound `0.0.0.0:9001` (it's the ingress hub + needs loopback for

> rebinding it. Only the two apps that are pure forward *targets* moved off the public NIC.

## Convention established (for future apps behind miaw-route)
Internal forward targets **bind `10.0.0.5`** (reachable by the cluster proxy for public
ingress *and* by same-host services like miaw-route; never on the public NIC) and are
**referenced as `http://10.0.0.5:PORT/...`** in routes. Apps with a configurable bind use a
`HOST` env var; self-referential health checks must use the bound host, not `localhost`.
