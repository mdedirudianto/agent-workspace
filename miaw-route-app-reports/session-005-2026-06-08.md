## Goal

The default (catch-all) route forwarded unmatched webhooks to a **webhook.site** URL.
webhook.site has free-tier limits (rate/expiry), so the fallback was unreliable ‚Äî
forwards to it intermittently failed. Replace it with a **local in-app echo sink** so
the catch-all always lands somewhere we control and returns 200.

> Note: payloads were never actually *lost* ‚Äî `webhook_logs` captures every forward
> (incl. failures) before/at forward time ([webhook.ts:351](https://github.com/biji-dev/miaw-route/blob/main/src/routes/webhook.ts)).
> The win is a reliable, always-200 sink (no external dependency) and clean `success`
> status instead of webhook.site failures.

## The echo endpoint ‚Äî [`src/routes/echo.ts`](https://github.com/biji-dev/miaw-route/blob/main/src/routes/echo.ts)

Dummy test sink, in-memory ring buffer (last 50), **no DB, no signature check**:
- `POST /echo` ‚Üí record + return `{ received: true, seq }`
- `GET /echo` ‚Üí buffered payloads newest-first (`{ count, entries }`)
- `DELETE /echo` ‚Üí clear buffer

### Security fix added this session (commit `8ab6ae0`)
nginx proxies `/echo` publicly (`location /` ‚Üí app), and once the default route forwards
**real prod WhatsApp payloads** into the buffer, `GET /echo` would leak them with no auth.
Added an `onBeforeHandle` guard: **`POST` stays open** (internal `localhost` forward),
**`GET`/`DELETE` require the admin API key** (reuses `checkApiAuth`, same as `/api/*`).
Also updated the seed default route (`Default Fallback` ‚Üí `Default Echo`,
`localhost:4000/webhook` ‚Üí `localhost:3000/echo`) for fresh-install consistency. Bumped to **v0.9.7**.

## Prod DB cutover (on `db`, `miaw_route`)

In-place UPDATE of the existing default row (keeps `id` `bb265177-‚Ä¶`, so existing
`webhook_logs.route_id` references stay intact):

```sql
UPDATE routes
SET name = 'Default Echo',
    backend_url = 'http://localhost:9001/echo'
WHERE is_default = true;
```

> ‚öÝÔ∏è **Route cache:** the matcher caches routes in-memory with a **60s TTL**
> ([matcher.ts:14](https://github.com/biji-dev/miaw-route/blob/main/src/services/matcher.ts)).
> The direct SQL UPDATE bypasses the app's `invalidateRouteCache()`, so the change is
> picked up on the next TTL refresh (‚â§60s) ‚Äî no restart required. (Editing the route via
> the dashboard/API would invalidate immediately.)

**Self-URL is `localhost:9001`** (the PM2 listen port), **not** the `localhost:3000` used
in the seed ‚Äî 3000 is the docker-internal port from `docker-compose.yml`, irrelevant to the
PM2 deploy.

## Notes / follow-ups

- The old webhook.site URL (`674344ee-‚Ä¶`) is now unreferenced ‚Äî nothing to clean up
  externally (it just stops receiving traffic).
- Echo buffer is **per-process, in-memory** ‚Äî resets on restart, not shared across HA
  replicas. Fine for a debug sink; don't treat it as durable capture (that's `webhook_logs`).
- To inspect what the catch-all is capturing: `curl localhost:9001/echo -H "x-api-key: <admin key>"`
  on `app` (or via the public URL with the same header).
