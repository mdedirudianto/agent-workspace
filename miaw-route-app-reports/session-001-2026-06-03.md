## What miaw-route does

WhatsApp Cloud API webhook router. Meta only allows one webhook URL per app — miaw-route receives all webhooks and routes them to multiple backends based on phone number ID, WABA ID, or regex pattern. Stack: Bun/Elysia, PostgreSQL (routes + logs), Redis (retry queue + sessions).

---

## Deployment Decision

Chose **app server** (not proxy server) for the following reasons:

| Option | Decision | Reason |
|--------|----------|--------|
| App server `154.26.129.104` | ✅ Chosen | Follows cluster pattern; Docker already present; Fluent Bit already configured; production `nginx.conf` in repo was pre-written for this topology |
| Proxy server `46.250.234.153` | ❌ Rejected | Proxy is a dumb gateway — app logic + DB/Redis there risks destabilising all cluster traffic |
| PM2 (no Docker) | ✅ Chosen | Dev server already uses PM2 + Bun; no isolation benefit since DB/Redis are already external on `db` server |

## Gotchas Encountered

| Issue | Root Cause | Fix |
|-------|-----------|-----|
| PM2 `env_file` not loading | PM2's `env_file` directive requires dotenv — not built into PM2 itself | Used `interpreter_args: '--env-file=/path/.env.production'` instead — Bun's native `--env-file` flag |
| 502 from proxy after start | `HOST=127.0.0.1` in `.env.production` bound app to loopback only; proxy on `10.0.0.5` couldn't reach it | Changed to `HOST=0.0.0.0`; restarted with `pm2 restart miaw-route --update-env` |
| HTTPS 526 from Cloudflare | Cert didn't exist yet on origin; CF in Full/strict mode | Expected — resolved after certbot ran |
| HTTPS clone 403 | Repo is private; HTTPS needs a token | Used SSH remote `git@github.com:biji-dev/miaw-route.git` (key already authorised) |
