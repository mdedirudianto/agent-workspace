Both apps declare `output: "standalone"` in `next.config.ts` but were being run with `bash -c npm start` (i.e. `next start`), which Next.js 16 explicitly refuses (`⚠ "next start" does not work with "output: standalone" configuration`). Switch both to `node .next/standalone/server.js`. **This becomes the standard for every Next.js deploy going forward.**

---

## The standard (apply to every Next.js app from here on)

Per-deploy procedure for any Next.js app declaring `output: "standalone"`:

1. **`.env`** must include `HOSTNAME` (in addition to or instead of `HOST`). The standalone `server.js` reads `process.env.HOSTNAME` (default `0.0.0.0`) and `process.env.PORT` — it ignores `HOST`. Keep `HOST` if other code reads it; add `HOSTNAME` next to it.
2. **Build:** `nohup bash -c 'npm run build > /tmp/<app>-build.log 2>&1; echo BUILD_EXIT=\$? >> /tmp/<app>-build.log' &` then wait for `BUILD_EXIT=0`. Never run `next build` in a foreground SSH session (see session-001 F2).
3. **After build, copy assets into the standalone tree:**
   ```
   cp -r .next/static .next/standalone/.next/static
   cp -r public       .next/standalone/public
   ```
   `next build --output=standalone` does **not** include these. Without the copy, `/_next/static/*` and `/favicon.ico` (etc.) return 404 and the page renders broken.
4. **PM2 entry:**
   ```
   pm2 delete <name>
   cd /home/devops/<dir>
   pm2 start ./node_modules/.bin/dotenv \
       --name <name> --interpreter node \
       --cwd /home/devops/<dir> \
       -- -e .env -- node .next/standalone/server.js
   pm2 save
   ```
   `dotenv-cli` (already in each repo's `node_modules/.bin/`) loads `.env` since the standalone server.js doesn't.
5. **Verify:** internal `curl http://10.0.0.5:<port>/` → 200; internal `/_next/static/<one-asset>` → 200; external `https://<domain>/` → 200; `pm2 list` restart counter `↺ 0` after a minute.

The stale `ecosystem.config.js` in each repo is **wrong** (ports don't match, no HOSTNAME, no `.env` loading) — ignore it. Cleaning it up is a repo-level change, not deploy work.
