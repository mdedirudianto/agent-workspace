## Findings / Gotchas

| Tag | Finding | Note |
|-----|---------|------|
| F1 | **`NEXT_PUBLIC_*` is build-time** | Restart-only would leave canonical, og:url, WhatsApp dashboard links, etc. pointing at the old domain. Anyone rotating an app URL on a Next.js box must rebuild, not just `pm2 restart`. |
| F2 | **Don't run `next build` in a foreground SSH** | When the SSH wrapper detached mid-compile, the build was killed but `.next/BUILD_ID` was already on disk — leaving a half-built directory that PM2 happily served and that crash-looped on missing turbopack chunks. Always `nohup`-ish the build (`nohup bash -c 'npm run build > /tmp/<app>-build.log 2>&1; echo BUILD_EXIT=$? >> /tmp/<app>-build.log' &`) and poll for the exit marker. |
| F3 | **"`next start` does not work with `output: standalone`" warning is non-fatal (for now)** | Both `pasarbesar` and `pasarbesar-lpekinhub` PM2 entries run `bash -c npm start` (i.e. `next start`) despite `next.config.ts` declaring `output: "standalone"`. The warning prints on every startup but the server still serves traffic. The proper layout would be `node .next/standalone/server.js`, matching what `ecosystem.config.js` already documents. Worth aligning in a future session — not urgent. |
| F4 | **Cloudflare in front of proxy** | `pasarbesar.id` (and now `lpekinhub.pasarbesar.id`) terminate TLS at Cloudflare, then re-establish to our origin using our Let's Encrypt cert. `openssl s_client` against the public hostname shows the CF edge cert; SAN inspection has to happen against `proxy:127.0.0.1:443` with SNI to see our origin cert. |
| F5 | **Old `lpekinhubjatim.faazdigi.com` left dead, not redirected** | Per user direction: keep it parked. DNS still resolves to `2.57.91.92` (not ours), proxy has no config, certbot has no cert. Nothing to do unless it's reactivated. |
