## Changes (two PRs, both merged `--no-ff` to `main`)

**PR [#2](https://github.com/biji-dev/seenow/pull/2)** `feat(api): bind-address env + env-driven wildcard CORS`
- **`API_HOST`** — optional bind address passed to `@hono/node-server` `serve()` (`apps/api/src/server.ts`).
  Empty = `0.0.0.0` (unchanged). Set to the private IP on cluster hosts so the API is reachable only via
  the proxy/local nginx, never the public NIC.
- **`TENANT_ROOT_DOMAIN`** — `corsOrigin(env)` replaces the fixed `webOrigins` array in the hono `cors`
  middleware: allows any exact `WEB_ORIGINS` entry **plus** any `https://` origin at that root or a
  subdomain of it. Empty = fixed list only.

**PR [#3](https://github.com/biji-dev/seenow/pull/3)** `fix(auth): trust https://*.<TENANT_ROOT_DOMAIN>`
- Found while verifying #2: hono CORS allowed tenant origins, but **Better Auth's own `trustedOrigins`**
  still only trusted `WEB_ORIGINS`, so tenant sign-in 403'd *beneath* CORS. Added `trustedOrigins(env)`
  (= `webOrigins` + `https://*.<root>` when set) and wired it into `apps/api/src/auth/better-auth.ts`.

Tests: `apps/api/test/cors-origin.test.ts` covers both matchers (exact + wildcard-subdomain allow;
http/foreign/look-alike/empty reject). **typecheck 2/2 · tests 6/6.**

- **Bind:** `ss` confirms prod `10.0.0.5:4900`, dev `127.0.0.1:4900` (no longer `0.0.0.0`). Proxy/local

  with a seeded user → **200** + `Set-Cookie …; Domain=.dev.seenow.ac; Secure` + `access-control-allow-origin:
  https://acme-corp.dev.seenow.ac`. End-to-end: a tenant subdomain can now authenticate.
