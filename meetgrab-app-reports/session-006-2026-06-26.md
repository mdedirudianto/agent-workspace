## Bug fixed: OpenPanel URL was wrong since session-001

Session-001 baked `PUBLIC_OPENPANEL_API_URL=https://analytics.biji.uk`. The web `sendOpenPanel()` appends `/track`, so calls were going to `https://analytics.biji.uk/track` — a path that does not exist on the self-hosted OpenPanel instance. The `@openpanel/sdk` default in `apps/api/src/env.ts` uses `https://analytics.biji.uk/api` as the base, so the correct endpoint is `/api/track`. This session bakes in the correct value.

| Var | Session-001 | This session |
| --- | --- | --- |
| `PUBLIC_OPENPANEL_API_URL` | `https://analytics.biji.uk` → hit `/track` ❌ | `https://analytics.biji.uk/api` → hit `/api/track` ✓ |
