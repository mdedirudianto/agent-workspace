# meetgrab App Reports

App-scoped reports for the **meetgrab** product (`meetgrab.us`) ‚Äî a Chrome extension that turns Google Meet captions into AI meeting summaries, backed by a stateless LLM-proxy API and an Astro marketing site.

**Repo:** `github.com/biji-dev/meetgrab` (pnpm + turbo monorepo). Apps: `api` (Hono/Node), `web` (Astro static), `admin` (Vite SPA ‚Äî **not deployed yet**), `extension` (Chrome Web Store ‚Äî not server-deployed). Packages: `shared-types`, `prompts`, `ui`.

**Server footprint:**
- `app` (`154.26.129.104` / `10.0.0.5`) ‚Äî `meetgrab-api` under **PM2** (Hono on Node), **private-bound `10.0.0.5:8930`**, `node --env-file`. Source at `~/meetgrab` (devops). **No Docker** (repo ships a Dockerfile/compose for a DO droplet; we run native PM2 per app-server convention).
- `proxy` (`46.250.234.153`) ‚Äî public nginx; terminates TLS for `meetgrab.us`, `www`, `api.meetgrab.us`. Serves the Astro static build from `/var/www/meetgrab.us`; reverse-proxies `api.` ‚Üí `10.0.0.5:8930`. CF orange-cloud ‚Üí Full(strict). LE cert via DNS-01 (`/etc/letsencrypt/cloudflare-meetgrab.ini`).
- `monitor` (`10.0.0.3`) ‚Äî GlitchTip (`errors.biji.uk`, API project **14**), OpenPanel (`analytics.biji.uk`, shared **MeetGrab Web** project), OpenObserve (logs via `app` fluent-bit glob).
- `db` (`10.0.0.1`) ‚Äî **Postgres `meetgrab` DB** (session-007): `uninstall_feedback`, `usage_daily`, `usage_user` tables. User `meetgrab`.

**Architecture note (subdomain, not `/api`):** the API is a separate subdomain `api.meetgrab.us`. The extension ships pointing at it; CORS in `apps/api/src/app.ts` whitelists `chrome-extension://<32-char>` + `https://meetgrab.us` only. The marketing site does **not** call the API.

**Current deployed SHA:** api + web **`ac90a6c`** (main, 2026-07-14, api **v0.10.0** ‚Äî adaptive LLM escalation to Sonnet 4.6 for oversized transcripts; web **v0.11.0** ‚Äî founding-user offer / Free-Pro pricing section). Deployed session-010.

**Auth / LLM:** primary = Anthropic **Claude Sonnet 4.6** (`claude-sonnet-4-6`, switched session-007) via **OAuth** subscription token (reused from dogfooding; `ANTHROPIC_AUTH_MODE=oauth`, static `sk-ant-oat01-‚Ä¶`, **no refresh token** ‚Üí may expire, watch for 401s). Fallback = **Gemini 2.5 Flash** (`GOOGLE_AI_API_KEY`). **Adaptive escalation (session-010):** `LLM_LARGE=anthropic:claude-sonnet-4-6` ‚Äî transcripts too big for the primary model's window auto-escalate instead of failing `TRANSCRIPT_TOO_LARGE`. Cost circuit-breaker `DAILY_SPEND_CAP_USD=20`; rate limit 30/IP/day.

**Observability env:** GlitchTip API DSN `‚Ä¶@errors.biji.uk/14` ‚úì; **web GlitchTip DSN `‚Ä¶@errors.biji.uk/16`** (project `MeetGrab-Web`, org GrabFamily/team TheGrab) ‚úì; OpenPanel server-side reuses the **MeetGrab Web** project's write client (`d524407b‚Ä¶` + secret) ‚úì; web GA4 `G-KMVL074MH2` + OpenPanel `analytics.biji.uk` baked into build ‚úì; OpenObserve auto-ships meetgrab-api PM2 JSON logs via the existing `app` fluent-bit glob (`default/app` stream) ‚úì.

**Uptime-Kuma:** #47 `meetgrab.us` (http), #48 `MeetGrab API Health` (keyword `"ok":true` on `/v1/health`), #49 `MeetGrab Summary Canary` (push, push-token monitor; cron `~/meetgrab-summary-canary.sh` on `app` at 07:00/19:00 does a real `POST /v1/summaries`).
