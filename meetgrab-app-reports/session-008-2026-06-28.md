## Canary fix

The summary canary (`~/meetgrab-summary-canary.sh`) was checking for `HTTP 200 + "summary"` in the response body. The async rewrite in session-005 changed the contract to `HTTP 202 + jobId → poll until done`. As a result the canary had been falsely reporting **DOWN** to Uptime-Kuma #49 on every run since session-005 (2026-06-24).

Rewrote the script to:
1. `POST /v1/summaries` → expect 202, extract `jobId`
2. Poll `GET /v1/summaries/:jobId` every 3s up to 90s timeout
3. Report UP only on `state: done`; report DOWN on failed/timeout/curl-fail
