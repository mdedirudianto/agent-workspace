**Date:** 2026-06-22

**Repo:** `github.com/biji-dev/meetgrab` @ `main` `6e2cfb5` (api v0.4.1, web v0.3.2)

## Goal

Deploy the new meetgrab app to production. DNS for `meetgrab.us`, `www`, `api` was already pointed at Cloudflare (orange-cloud ‚Üí proxy). Decide subdomain vs `/api`, wire env + observability, no DB unless required.

## Key analysis findings

| Finding | Implication |
| --- | --- |
| API (`@meetgrab/api`) is a **stateless Hono/Node LLM proxy** ‚Äî no Postgres/Redis/Mongo; rate-limit is in-memory per-IP-hash | **No `db` work at all** |
| Repo ships nginx configs for **`api.meetgrab.us` subdomain** + CORS in code whitelists `chrome-extension://‚Ä¶` + `https://meetgrab.us` | **Recommendation: subdomain, not `/api`** ‚Äî already built that way, DNS ready |
| Repo's OPS assumes a **single DigitalOcean droplet + Docker** | Adapted to our cluster (CF ‚Üí proxy ‚Üí app private) and **PM2, not Docker** (app-server convention) |
| `web` is **Astro static** (no SSR adapter) and does **not** call the meetgrab API | Served as static files on proxy `/var/www/meetgrab.us` |
| `admin` is a Vite SPA, no DNS pointed; `extension` ‚Üí Chrome Web Store | Both **out of scope** for this deploy |
