## Actions Taken

| # | Item | Command | Recovered |
|---|------|---------|-----------|
| 1 | Xcode DerivedData | `rm -rf ~/Library/Developer/Xcode/DerivedData` | 10GB → 0 |
| 2 | Docker installer leftover | `rm -rf ~/Library/Application\ Support/com.docker.install` | 2.1GB → 0 |
| 3 | 10 non-booted simulator devices | `xcrun simctl erase <uuid>` ×10 (iPhone 16e left Booted/untouched) | 3.9GB → 1.8GB |
| 4 | Docker dangling images/volumes | `docker system prune -f` + `docker volume prune -f` | 0B (nothing unreferenced found) |
| 5 | Homebrew old versions | `brew cleanup` | ~627MB |
| 6 | Android AVD snapshots | `rm -rf ~/.android/avd/Pixel_6_API_34.avd/snapshots` | 1.3GB → 0 |
| 7 | All project `node_modules` (72 dirs across eventopia, grobiz, meetgrab, KvernaProject, Playground/pixart-chargen, Playground/chargen) — `numa-home` explicitly excluded | `rm -rf` per dir | 4GB actual (see finding below — du summed to ~8.3GB) |

Approved in three rounds: Tier A+B (items 1–3) approved first given the active-booted-device nuance; items 4–6 approved as a follow-up "deeper sweep"; item 7 approved after presenting a per-project size/staleness table.

**Finding — pnpm hardlink store means `du` overstates node_modules reclaim:** eventopia, grobiz, meetgrab, and KvernaProject/kverna-web all use `pnpm-lock.yaml`. pnpm installs via hardlinks from the content-addressable `~/Library/pnpm/store` (8.5GB, unchanged by this deletion — verified via `pnpm store status`), so `du` on their `node_modules` counts bytes that are *also* counted in the store. Deleting the node_modules only frees the directory-entry/hardlink, not the underlying blocks (those stay alive as long as the store references them). Real recovery here (~4GB) came mostly from `Playground/chargen` + `Playground/pixart-chargen` (no lockfile — real copies, not hardlinked) plus non-hardlinked build artifacts. **To reclaim the rest, `pnpm store prune` is now the correct next step** — with eventopia/grobiz/meetgrab/kverna's node_modules gone, a meaningful chunk of the 8.5GB store may now be unreferenced.

| 8 | pnpm store prune | `pnpm store prune` | 8.5GB → 4.9GB store (3.6GB freed); removed 2,280 packages / 98,834 files |
