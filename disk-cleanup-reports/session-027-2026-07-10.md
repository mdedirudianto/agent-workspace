# Session 27 — Critical Near-Full Recovery + First Deep Clean of `klo` Account

**Date:** 2026-07-10
**Disk before:** 97% used — 7.4GB free
**Disk after:** 92% used — 17GB free
**Recovered:** ~10GB

---

## Context

Disk had regrown to critical levels overnight after Session 26 ended at 32GB free (86%) last night — down to 7.4GB free (97%) by this session. While investigating, macOS was observed auto-purging purgeable caches on its own mid-session (CocoaPods 843MB and `ms-playwright` 1.0GB vanished between two consecutive `du` checks as free space ticked up from 7.4GB→9.4GB) — a sign the OS starts reclaiming purgeable cache space automatically once free space gets this tight.

This session had two phases: (1) the standard `dedi`-side repeat-offender sweep, and (2) at the user's request, the first-ever deep clean of the `klo` automation account beyond the one-time Ollama orphan removal in Session 24 — this time targeting logs, browser cache, git bloat, and confirmed-unused local LLM model weights rather than just "regenerable caches."

### Findings

| Location | Size | Verdict |
|---|---|---|
| Xcode DerivedData | 3.4GB | Standard repeat offender |
| CoreSimulator devices | 2.4GB | Standard repeat offender |
| `~/.npm/_npx` + `_cacache` | 902MB | Fifth+ consecutive session needing manual clear |

| Docker images | 3.0GB, 0% reclaimable | All 4 attached to running grobiz stack |

### Actions Taken

| # | Item | Command | Recovered |
|---|------|---------|-----------|
| 1 | Xcode DerivedData | `rm -rf ~/Library/Developer/Xcode/DerivedData/*` | 3.4GB → 0B |
| 2 | CoreSimulator devices | `xcrun simctl shutdown all` then `xcrun simctl erase all` | 2.4GB → 192MB |
| 3 | npm caches | `rm -rf ~/.npm/_npx ~/.npm/_cacache` | 902MB → 52KB |
| 4 | dotslash cache | `rm -rf` (partial, permission denied on signed `.app`) then `sudo rm -rf ~/Library/Caches/dotslash` | ~365MB → 0B |
| 5 | Docker unused volumes | `docker volume prune -f` | 236MB |

Approved as a batch after the recommendation was presented. Result: 9.4GB → 18GB free (96%→91%).

## Phase 2 — `klo` account deep clean

### Findings

Unlike Session 24's `klo` review (which found one big orphaned app, `.ollama`, and otherwise left everything alone), this pass went looking for the equivalent of "node_modules bloat" specifically. None was found — `klo` is a pure automation account with no dev-project checkouts (`~/Projects` is empty) — but a single runaway log file turned out to be the dominant cost:

| Location | Size | Verdict |
|---|---|---|
| `.openclaw/logs/ttyd-err.log` | **2.04GB** | Single unrotated stderr log, actively held open by the running `ttyd` process (PID 313) |
| `.openclaw/logs/gateway.err.log` | 187MB | Same pattern, no process currently holding it open |
| `.openclaw/browser/openclaw/user-data` | 463MB | Chromium automation profile — `Cache` (178MB) + `Code Cache` (186MB) + `GraphiteDawnCache`/`GPUCache`/Service-Worker `CacheStorage` were pure cache; Cookies/IndexedDB/Sessions/Local Storage held the actual login state (e.g. Trello) and were left alone |
| `.openclaw/.git` | 191MB (184MB loose + 9.2MB packed) | Config/state history repo, never gc'd |
| `.cache/uv` + `.cache/prisma` | 133MB | Pure package-manager caches |
| `.yarn/berry` | 83MB | Yarn cache |
| `.node-llama-cpp/models` | 313MB | Local LLM model weights — **user confirmed unused** |
| node_modules search (maxdepth 6) | ~150MB total, 5 small dirs | All tool-internal (`.nvm`, Zed's bundled prettier/json-language-server, pnpm's own self-install, openclaw's bundled node) — no stray project installs, consistent with S24's addendum finding |
| `.openclaw/memory` + `memory/backups` | 327MB | 7 daily sqlite backups (Jul 4–10), self-rotating, not unbounded — real data, left alone |
| `.openclaw/tools` (sherpa-onnx-tts + bundled node) | 555MB | Runtime binaries openclaw needs to function — not removed without confirming it can redownload |
| `.local/pipx/venvs` | 916MB | Real installed Python tool environments, not caches — not simple cache cleanup, left for a future review |

### Actions Taken

| # | Item | Command | Recovered |
|---|------|---------|-----------|
| 1 | `ttyd-err.log` | `sudo truncate -s 0` (in place, no restart — ttyd's fd stays valid) | 2.04GB → 0B |
| 2 | `gateway.err.log` | `sudo truncate -s 0` | 187MB → 0B |
| 3 | Browser cache only | `sudo rm -rf` on `Cache`, `Code Cache`, `GraphiteDawnCache`, `GPUCache`, `Service Worker/CacheStorage` | user-data 463MB → 64MB |
| 4 | `.openclaw` git repo | `sudo git -C /Users/klo/.openclaw gc` | loose objects packed (~189MB overhead → ~147MB) |
| 5 | `.cache/uv` + `.cache/prisma` | `sudo rm -rf` | 139MB → 6.1MB |
| 6 | `.yarn/berry` | `sudo rm -rf` | 83MB → 0B |
| 7 | `.node-llama-cpp` | `sudo rm -rf` (user confirmed unused) | 313MB → gone |

All steps approved by user before execution. **`klo` home: 8.7GB → 5.6GB.**
