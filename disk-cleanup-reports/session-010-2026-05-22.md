## Why Disk Refilled Since Session 9

Only 3 days since S9 ended at 169GB used / 30GB free. Regrew ~17GB to 5.3GB free. **New top offender this time is not Docker — it's VM swap.**

| Source | Δ | Cause |
|--------|--:|-------|
| **VM swap** | **+14G** | 15 × 1GB swapfiles; `vm.swapusage` = 13.2G used / 14G total. Load avg ~6, 13 login sessions, 7-day uptime → sustained memory pressure spawning swapfiles |
| **Docker.raw** | +8G (9G → 17G) | grobiz/ytgrab/iccn dev — build cache 4.4G + extra images |
| node_modules | +misc | numa-home, ytgrab, asiaweek active |

The headline: this is a **RAM problem surfacing as a disk problem**. macOS keeps allocating 1GB swapfiles as memory pressure climbs. That's the 14G that wasn't there at S9.

---

## Actions Taken

### Tier 0 — Reboot (user-triggered, ~13–14G + clears thrash)

Swap was 13.2G of 14G in use (1.1G free). Swapfiles can't be `rm`'d while in use — a **reboot** regenerates them empty, reclaiming ~13–14G instantly and clearing the memory thrash. **Pending — user to trigger.** Underlying fix: close idle apps / trim the 13 concurrent login sessions; the machine is over-subscribed on RAM.
