## Actions Taken

### Restart (user-initiated, before this session's manual work)
Recovered **~13G** (3.1GB → 18GB free). Cleared VM swap (now 0), purgeable space, and the 2.2G stale Docker installer `in_progress/` (Docker cleans it on launch). System volume reading is the sealed snapshot; real pressure is on `/System/Volumes/Data` (`disk3s5`).

### Tier 1 — Safe build/cache sweep (~5G)

| # | Item | Command | Recovered |
|---|------|---------|-----------|
| 1 | **All `.next` build caches** | `find ~/Projects -maxdepth 4 -type d -name .next -exec rm -rf {} +` | ~4.3G |
| 2 | **VSCode ShipIt update cache** | `rm -rf ~/Library/Caches/com.microsoft.VSCode.ShipIt` | 1.1G |

Net 18GB → **23GB free**. Nothing running was touched; both regenerate on demand.

### Tier 2 — All node_modules + pnpm store (~10G)

User opted to clear **all** `node_modules` (reinstall on demand) plus the global pnpm store.

| # | Item | Command | Recovered |
|---|------|---------|-----------|
| 1 | **All `node_modules`** (every project, all workspaces) | `find ~/Projects -type d -name node_modules -prune -exec rm -rf {} +` | ~8G |
| 2 | **pnpm global store prune** | `pnpm store prune` — removed 2538 packages / 141039 files | ~2G |

Net 23GB → **33GB free**. Reinstall with `pnpm install` per project when work resumes.

### Tier 3 — Docker reclaim (user-approved, ERPNext excluded)

User restarted Docker (clean stop → APFS compaction shrank `Docker.raw` **13G → 10G** on host, ~3G). Then, daemon back up:

| # | Item | Command | Reclaimed |
|---|------|---------|-----------|
| 1 | Stopped containers + stale networks | `docker system prune -f` | 627M (in-VM) |
| 2 | Unused images | `docker image prune -af` | 520M (in-VM) |
| 3 | **5 dangling volumes** (by name) | `docker volume rm 928ae3… eventopia_pg iccn_openclaw_node_modules iccn_openclaw_state infra_eventopia_pg` | ~2G (in-VM) |

⚠️ **ERPNext explicitly excluded** — `grobiz_erpnext-{apps,logs,sites}` + `grobiz_mariadb-data` are active and were never touched. Deleted volumes were eventopia/iccn local test data (regenerable).

⚠️ The ~3G freed inside the VM by steps 1–3 is now slack in `Docker.raw`; it returns to the host on the **next** Docker restart (compaction). Only the 3G host-shrink (13→10G) landed this session.

Net host disk after all tiers: **36GB free / 83%**.
