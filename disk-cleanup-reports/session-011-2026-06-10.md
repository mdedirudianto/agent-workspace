| Source | Δ | Cause |

| **Chrome OptGuide (new variant)** | **+4G** | New folder `OptGuideOnDeviceModel/` (Gemini Nano on-device model, v2025.8.8.1141). The S8 enterprise policy locked `optimization_guide_model_store` (held at 179M) but a **different** model dir grew to 4G. |

| **turbo caches** | +2.5G | eventopia/.turbo alone was 2.2G |

**Swap is NOT an offender this session** — machine was rebooted ~8 min before this run (uptime 8m), so `vm.swapusage` = 0. The S10 swap blowout root cause (RAM pressure) is dormant, not fixed; watch on long uptimes.

### Tier 2 — Docker (~5G, daemon was down — started it)

Docker daemon was DOWN (Desktop process present but not serving). Started `Docker.app`, waited ~10s for daemon, then:

| Item | Command | Recovered |
|------|---------|-----------|
| Build cache | `docker buildx prune -af` | **3.75G** |
| Unused images | `docker image prune -af` | 319M |
| Containers/networks | `docker container/network prune -f` | minor |

**Docker.raw auto-shrank 18G → 13G** on APFS post-prune (no manual compaction, consistent with S10).
