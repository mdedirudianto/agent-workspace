## Context

User had recently updated macOS (now 26.5.2, build 25F84) and asked for a full disk analysis plus any post-update residual files. Checked all the usual OS-update leftover locations — none were an issue:

| Location checked | Result |
|---|---|
| `/System/Volumes/Update` | Clean — only small logs/plists (<4MB total) |
| `/Library/Updates` | 592KB, nothing to clean |
| Local Time Machine snapshots (`tmutil listlocalsnapshots`) | None on `/` or `/System/Volumes/Data` |
| `/private/var/folders` (system temp) | 162MB, normal |
| `Downloads` for leftover `.dmg`/`InstallAssistant`/installer packages | None found (only a small `RealVNC Connect Setup.app`, 4.2MB) |

The macOS update itself left no residue. The real disk pressure was the usual dev-tool cache accumulation plus a new offender: **Xcode Simulators**.

## Actions Taken

| # | Item | Command | Recovered |
|---|------|---------|-----------|
| 1 | CoreSimulator device content (11 devices: iPhone 17/17 Pro/17 Pro Max/Air/16e, iPad Pro M5 x2, mini/Air/base — all Shutdown) | `xcrun simctl erase all` | **~9.7GB** (9.9GB → 192MB) — device *definitions* kept, only installed apps/photos/simulated data wiped |
| 2 | Chrome Service Worker CacheStorage/ScriptCache + GPUCache + Dawn caches (Default profile) | `rm -rf` targeted subfolders | ~575MB — regenerates automatically, no login/bookmark/password loss |
| 3 | Misc tool caches | `rm -rf ~/.cache/{node,prisma,opencode,selenium}` | ~180MB |
| 4 | Docker system prune | `docker system prune -f` | 86KB — only 4 stopped containers + 2 unused networks; nothing else dangling |
| 5 | pnpm store prune | `pnpm store prune` | 0B — 0 packages orphaned, all 4.8GB fully referenced by active projects |
| 6 | npm cache clean | `npm cache clean --force` | Evicted, size negligible this round |
