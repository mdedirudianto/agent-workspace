## Root Cause Analysis — Why Disk Fills So Fast

| Source | Regrowth rate | Recommended fix |
|--------|--------------|-----------------|
| **Claude Desktop VM** | 10G per app launch | Delete `vm_bundles/`; VM re-expands only if agent mode enabled |
| **macOS Aerial screensaver** | 4–6G per macOS update | System Settings → Screen Saver → switch away from Aerials |
| **VS Code auto-update (ShipIt)** | ~800M per update | Can't prevent; flush monthly with `rm -rf ~/Library/Caches/com.microsoft.VSCode.ShipIt` |
| **Chrome cache** | Hundreds of MB per hour browsing | Chrome Settings → Privacy → Clear cache periodically |
| **pnpm / npm metadata** | ~300M on first install after clean | Acceptable; unavoidable |
| **Multiple editors** (VS Code + Cursor + Zed + Antigravity + Trae) | Each holds independent caches totaling 2–4G | Pick one primary editor, uninstall the rest |
