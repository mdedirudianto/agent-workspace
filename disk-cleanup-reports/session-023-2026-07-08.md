# Session 23 — Same-Day Mobile Build Regrowth (Gradle/DerivedData/Simulator/npx)

**Date:** 2026-07-08
**Disk before:** 93% used — 18GB free
**Disk after:** 88% used — 27GB free
**Recovered:** ~9GB

---

## Context

User asked for a general cleanup, "just like usual." Disk had dropped from 33.2GB free (S22, same day) to 18GB free within ~7 hours — the fastest same-day regrowth seen yet. Investigation via file mtimes showed the drop wasn't ongoing regrowth (no files touched in the prior 6 hours) but a one-time spike: today's grobiz mobile buildout (263-commit catch-up, mentioned in AM session notes) triggered Android (Gradle) and iOS (Xcode) builds sometime between S22's cleanup and ~13:07, regenerating the same caches S22 had just cleared.

---

## Actions Taken

| # | Item | Command | Recovered |
|---|------|---------|-----------|
| 1 | Gradle caches | `rm -rf ~/.gradle/caches` | 4.1GB → 0B |
| 2 | Xcode DerivedData | `rm -rf ~/Library/Developer/Xcode/DerivedData/*` | 2.8GB → 0B |
| 3 | CoreSimulator device content | `xcrun simctl erase all` | 2.5GB → 192MB (device definitions kept) |
| 4 | npm cache | `npm cache clean --force` | Negligible (didn't touch `_npx`) |
| 5 | pnpm store prune | `pnpm store prune` | 0B (no orphans) |
| 6 | npx cache | `rm -rf ~/.npm/_npx` | ~797MB — found after step 4 barely moved the needle; not part of the original plan, added mid-session |

Total measured recovery (18GB → 27GB free) came in under the ~10GB estimate from summing individual cache sizes, likely due to APFS space accounting / snapshot overhead rather than any step failing — all 6 commands completed without error.
