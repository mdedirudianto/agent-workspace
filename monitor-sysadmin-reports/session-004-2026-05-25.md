## Implementation Notes

### OpenObserve Custom SQL — key findings

1. **Metric chart**: requires a field dragged to Y-Axis via the `+Y` button in the Fields panel. The `SELECT COUNT(*) as count` query works; set Decimals → 0 in Config.

2. **Pie chart**: X-Axis must be set to the **grouping field** (e.g. `code`) via `+X`. The default `x_axis_1` auto-slot uses a time axis and shows "No Data" for categorical data. Remove it first, then add the field.

3. **Line chart (time series)**: The X-Axis slot is named `x_axis_1` internally. Alias the histogram column to `x_axis_1` in SQL so OO maps it directly:
   ```sql
   SELECT histogram(_timestamp, '1 hour') as x_axis_1, COUNT(*) as count
   FROM "nginx" GROUP BY x_axis_1 ORDER BY x_axis_1
   ```
   Using any other alias (e.g. `ts`) requires manually removing `x_axis_1` and adding the field — but even then the chart may not render. Aliasing to `x_axis_1` is the reliable approach.
   Note: aliasing to `_timestamp` is explicitly **not allowed** ("Alias '_timestamp' is not allowed" error).

4. **Table chart**: uses "First Column" (X-Axis) + "Other Columns" (Y-Axis). Remove the default `x_axis_1` time slot; add grouping field to X and count to Y.

5. **Bar charts**: same pattern as pie — remove `x_axis_1`, add grouping field to X, count to Y. Stream must be switched via the Stream dropdown in Fields panel.

6. **Time range**: Panels must be built at 1d (Past 1 Days) to see data — most Fluent Bit deployments only have a few hours of history. Dashboard default view resets to 15m; change at top-right.

7. **Stream switching**: After changing the stream dropdown, re-apply the query to refresh the Fields panel with the new stream's schema.
