## Implementation Notes

- **Alert name constraint**: OO alert names cannot contain spaces, `#`, `?`, `&`, `%`, or quotes. Use underscores (`SSH_Brute_Force_Burst`).
- **Threshold for SSH**: 53 Failed SSH events in 5 minutes was seen immediately â€” server is under active brute-force scanning. Threshold of 20 will fire frequently; raise to 50+ if noise is too high.
- **Nginx 5xx note**: The `code` field in the nginx stream is a string, so `code >= '500'` does a lexicographic comparison. This catches `500`, `502`, `503`, `504` correctly since all start with `5`.
- **App stderr threshold**: 100 stderr/5 min was not met at time of setup; may need tuning after baseline is established. Check `App Errors by Service` panel in Fleet Overview to calibrate.
- **Cooldown**: All alerts have 60-minute cooldown to prevent notification storms during sustained incidents.
- **str_match**: OO's `str_match(field, 'substring')` function is used for substring matching in SQL alert queries (equivalent to `LIKE '%substring%'`).
