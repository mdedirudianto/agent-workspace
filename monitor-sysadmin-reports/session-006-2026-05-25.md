## Fluent Bit Config Change Pattern

Added to each server's `/etc/fluent-bit/fluent-bit.conf`:

```ini
[INPUT]
    name              systemd
    tag               syslog.{server}
    Systemd_Filter    _SYSTEMD_UNIT=ssh.service
    Systemd_Filter    _SYSTEMD_UNIT=fail2ban.service
    Systemd_Filter    _TRANSPORT=kernel
    DB                /var/lib/fluent-bit/syslog-{server}.db
    Strip_Underscores On
    Read_From_Tail    On

# (only for app and db — other servers have a catch-all record_modifier filter)
[FILTER]
    name   record_modifier
    match  syslog.{server}
    record server {server}
    record host   {server}

[OUTPUT]
    name        http
    match       syslog.{server}
    host        10.0.0.3
    port        5080
    uri         /api/default/syslog/_json
    format      json
    http_user   admin@biji.uk
    http_passwd <redacted>
    tls         off
    compress    gzip
```

## Record Shape in OpenObserve

```json
{
  "_timestamp": 1779720190083000,
  "server": "proxy",
  "host": "proxy",
  "systemd_unit": "ssh.service",
  "transport": "syslog",
  "message": "Disconnected from authenticating user root 1.2.3.4 port 54321 [preauth]",
  "syslog_identifier": "sshd",
  "priority": "6"
}
```

Key fields: `server` (which machine), `systemd_unit` (`ssh.service` / `fail2ban.service`), `transport` (`syslog` or `kernel`), `message`.
