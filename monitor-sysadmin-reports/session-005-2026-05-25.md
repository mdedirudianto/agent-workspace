# Session 5 — OpenObserve Stream Retention Audit

**Date:** 2026-05-25
**Server:** `root@monitor` (`217.15.162.56` public, `10.0.0.3` internal)
**Goal:** Audit and explicitly set data retention for all 8 OpenObserve log streams.

---

## Finding: Global Default is 3650 Days

OpenObserve ships with a global default retention of **3650 days** (~10 years). No streams would have been silently purged. However, retention had never been explicitly set per-stream — values were invisible in the UI and would drift if the global default changed.

Previous sessions (backup session-002, numa session-003, dev session-002) claimed to set per-stream retention via `PUT /api/default/streams/{stream}/settings`, but verification via `GET /api/default/streams?type=logs` showed `"data_retention": 0` for all streams. This was a **read endpoint bug**: the list endpoint returns stale/aggregated data. The correct read endpoint is `GET /api/default/streams/{stream}/schema?type=logs`, which returns the live per-stream value.

## API Notes

- **Write endpoint:** `PUT /api/default/streams/{stream}/settings` — body `{"data_retention": N}` — returns `{"code":200,"message":""}`. Works correctly.
- **Read endpoint (correct):** `GET /api/default/streams/{stream}/schema?type=logs` — returns full settings including live `data_retention`.
- **Read endpoint (misleading):** `GET /api/default/streams?type=logs` — returns `"data_retention": 0` for all streams regardless of actual value. Do not use for verification.
- **Settings-only GET:** `GET /api/default/streams/{stream}/settings` returns HTTP 405 (PUT-only endpoint).
