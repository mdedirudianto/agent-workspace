## Implementation Notes

- **Filter via Condition tab**: The `systemd_unit` filter for fail2ban and SSH used the "Condition" tab (not "List") because no events in the 15-minute window meant the List dropdown showed no options. Condition tab allows manual operator + value entry.
- **Save works cleanly**: The Save button (`[data-test="dashboard-panel-save"]`) works without any DOM manipulation as long as no fetch interceptor is installed on `window.fetch`.
- **"No Data" for 15-minute window**: Fail2ban and SSH events are infrequent; panels show "No Data" at 15m default but populate correctly at 1d or 7d range. This is expected behaviour.
